Allocate Driver

Acceptance Tests (Automated)

  • When allocating a unit 00932451 to the following driver "Desilva 1,Deane" and an odometer value of 300000 the result will be an error. Error occurs because I cannot allocate a driver from an unrelated client.
  • When allocating a unit 00962644 to the following driver "Lienberger, Tammy" and an empty odometer value the result will be an error. Error occurs because odometer reading needs to be entered.
  • When allocating a unit 00962644 to the following driver "Lienberger, Tammy" and an allocation back-date of 1-20-2013 the result will be an error. Allocation cannot be back-dated .
  • When allocating a unit 00962644 to the following driver "Lienberger, Tammy" and an allocation forward-date of 1-20-2099 the result will be an error. Allocation cannot be forward-dated .
  • When allocating a unit 00962644 to the following driver "Lienberger, Tammy" and a deallocation back-date of 1-20-1901 the result will be an error. The current driver's deallocation date has to be greater than the allocation date.

Manual Tests

  • UOM is defaulted to MILES or KM, depending on the unit of measure found on the unit's odometer table value.
  • The From Date of the new driver must be the day after the To Date of the old driver.
  • A success test case. Allocating and deallocating a unit for a valid allocate/deallocate.
